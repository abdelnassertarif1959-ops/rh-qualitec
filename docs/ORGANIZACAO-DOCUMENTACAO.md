# Organização da Documentação

Este documento descreve a estrutura de organização da documentação do Sistema RH Qualitec.

## Estrutura de Pastas

### 📁 `/docs`
Pasta raiz da documentação do projeto.

### 📁 `/docs/builds`
Documentação relacionada a builds e deploys do sistema.
- Relatórios de build
- Status de compilação
- Problemas e soluções de build

### 📁 `/docs/seguranca`
Documentação de segurança, auditorias e correções de vulnerabilidades.
- Auditorias de segurança
- Implementação de JWT
- Correções de vulnerabilidades
- Relatórios de segurança

### 📁 `/docs/avisos`
Documentação do sistema de avisos e comunicados.
- Implementação do sistema de avisos
- Correções de bugs
- Integrações com dashboard
- Sistema de comentários
- Notificações

### 📁 `/docs/holerites`
Documentação do sistema de holerites.
- Correções de cálculos
- Melhorias de interface
- Filtros e visualizações
- Geração de PDF
- Período de referência

### 📁 `/docs/sessoes-trabalho`
Registros de sessões de trabalho e refatorações.
- Organizados por mês
- Resumos de trabalho realizado
- Refatorações importantes

## Convenção de Nomenclatura

Os arquivos seguem o padrão:
```
TIPO-DESCRICAO-DD-MM-AAAA.md
```

Onde:
- `TIPO`: CORRECAO, IMPLEMENTACAO, RESUMO, RELATORIO, etc.
- `DESCRICAO`: Breve descrição do conteúdo
- `DD-MM-AAAA`: Data da criação/modificação

## Arquivos Principais

- `PRD-SISTEMA-RH-QUALITEC-ATUALIZADO-2026.md`: Product Requirements Document completo
- `SISTEMA-*.md`: Documentação de sistemas específicos
- `COMO-*.md`: Guias de uso e tutoriais
- `TROUBLESHOOTING-*.md`: Guias de solução de problemas
