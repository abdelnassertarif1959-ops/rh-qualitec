## Categorias de Documentos

### Correções
Documentos que registram correções de bugs e problemas identificados.

### Implementações
Documentos que descrevem novas funcionalidades implementadas.

### Auditorias
Relatórios de auditoria de segurança e qualidade do código.

### Resumos
Documentos que consolidam informações de múltiplas sessões de trabalho.

### Guias
Documentos com instruções passo a passo para execução de tarefas.

## Como Usar

1. Para encontrar informações sobre uma data específica, busque pelos arquivos com a data no nome
2. Para encontrar informações sobre um tema específico, use a busca de texto nos arquivos
3. Os resumos são bons pontos de partida para entender o contexto geral

## Manutenção

- Novos documentos devem seguir o padrão: `TIPO-DESCRICAO-DD-MM-AAAA.md`
- Sempre adicione uma entrada neste README ao criar novos documentos
- Mantenha a organização cronológica
